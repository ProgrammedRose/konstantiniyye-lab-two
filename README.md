#  Прочитай меня ;)

## Установка и запуск приложения Docker
В терминале в корне проекта.

Сборка и запуск приложения:
`docker-compose up --build -d
`

Для остановки приложения
`docker-compose down`

Важно нормально указать путь до app, а то ошибки будут и ничего само собой не запустится :)

После запуска приложение будет доступно по адресу:
http://localhost:8000.

Swagger UI должен быть доступен по такому адресу:
http://127.0.0.1:8000/docs.

## Аутентификация API
Для работы с API через Swagger UI:

Нажмите кнопку Authorize

Введите:

    username: admin
    password: password (любой пароль подойдет)

## Слои проекта
Domain — реализация классов книг и покупок, тут же и ограничения (правила)

Application — сценарии использования (CRUD и оформление покупки)

Infrastructure — реализация хранения данных (сейчас это просто списки, 
которые существуют только в рамках текущей сессии)

Web (внешний слой) — REST API, Swagger, HTML


## Еще команды
Команды для работы с БД.

Просмотр логов приложения:
`docker-compose logs -f app`

Подключение к PostgreSQL для отладки:
`docker-compose exec db psql -U user bookstore`

Ручное применение миграций:
`docker-compose exec app alembic upgrade head`
