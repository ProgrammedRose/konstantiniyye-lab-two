#  Прочитай меня ;)

## Установка зависимостей и запуск приложения
В терминале:
`pip install fastapi uvicorn pydantic jinja2 python-multipart
`

Старт приложения (только из терминала):
`uvicorn src.app.main:app --reload
`.
Важно нормально указать путь до app, а то ошибки будут и ничего само собой не запустится :)

После запуска в консоли будет написан адрес, по 
которому приложение будет доступно.

Swagger UI должен быть доступен по такому адресу:
`http://127.0.0.1:8000/docs`
Через него следует создать несколько книг, а то их список будет пуст при старте программы.

## Слои проекта
Domain — реализация классов книг и покупок, тут же и ограничения (правила)

Application — сценарии использования (CRUD и оформление покупки)

Infrastructure — реализация хранения данных (сейчас это просто списки, 
которые существуют только в рамках текущей сессии)

Web (внешний слой) — REST API, Swagger, HTML

## Запуск и остановка проекта
Запуск:
`docker-compose up --build`

Остановка:
`Ctrl + C
`

Или в другом терминале:
`docker-compose down`

Полная очистка с удалением данных БД:
`docker-compose down -v`

Перезапуск:
`docker-compose down && docker-compose up --build`

## После успешного запуска
Приложение будет доступно по следующим адресам:

    Главное приложение: http://localhost:8000
    Документация Swagger UI: http://localhost:8000/api/docs
    Управление БД (PgAdmin): http://localhost:5050
